# Ecosystem Overview

## IronRidge Contractors

IronRidge Contractors is a fictional mid-sized commercial contractor and field operations company based in Indianapolis, Indiana. The business spans General Contracting, Field Services, Special Projects, and Operations Support, with real-world operational patterns centered on growth, coordination friction, and reporting gaps.

## The Four Demo Repositories

- `ops-visibility-demo` — executive reporting layer for project health, service backlog, closeout readiness, and risk
- `workflow-drag-reduction-demo` — internal support request workflow for intake, routing, approvals, and aging visibility
- `execution-infrastructure-demo` — lightweight execution tracker for commitments, blockers, ownership, and weekly review cadence
- `contractor-ops-system-demo` — field-aware operations tracker for issues, punch items, escalation, and closeout readiness

## How This Repository Fits

This repository is the most executive-facing part of the ecosystem. It turns shared operational records into concise dashboard outputs that leadership could review quickly without rebuilding the story by hand each week.

## Shared Company Context

All four repos use the same employee roster, project catalog, regions, divisions, and operating language so they feel like connected parts of one internal systems environment rather than separate demos.

## Portfolio Intent

This repo is meant to read as a clean portfolio skeleton: believable operating context, readable sample data, sharp documentation, and a screenshot-ready mock page that explains the concept quickly.
